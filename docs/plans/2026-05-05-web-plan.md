## Naming Note (Important)

The repo is called `web`. Inside it, `apps/web` is the **landing page** (riplai.id).  
These are two different things:

| | What it is |
|---|---|
| `web/` (repo root) | The entire Turborepo monorepo |
| `web/apps/web` | The public-facing landing page app |
| `web/apps/partner` | The partner PWA (inbox + analytics) |
| `web/apps/admin` | The admin panel |

## Before You Start ‚Äî Cross-Repo Contracts

### What you need from `api` before building each section

| Web section | Needs from api | Can build without? |
|---|---|---|
| Landing page | Nothing | Yes ‚Äî build now |
| Auth (login page) | `POST /auth/login` shape | Yes ‚Äî use fixture |
| Inbox list | `GET /conversations` shape | Yes ‚Äî use fixture |
| Conversation thread | `GET /conversations/:id` shape | Yes ‚Äî use fixture |
| Reply form | `POST /conversations/:id/reply` | Yes ‚Äî use fixture |
| SSE (real-time) | `GET /sse/events` running | No ‚Äî need api Phase 10 |
| Analytics stats | `GET /analytics/stats` | Yes ‚Äî use fixture |

### Mock strategy (build ahead of api)
No MSW needed ‚Äî keep it simple. Create `packages/api-client/src/__fixtures__/` with hardcoded JSON responses matching the expected api shapes. Swap `api-client/src/client.ts` to real Axios once api is ready.

```typescript
// packages/api-client/src/client.ts
// Toggle this constant while api is still being built:
const USE_FIXTURES = process.env.NEXT_PUBLIC_USE_FIXTURES === 'true'
```

### Type sharing
You write Go structs first in `api`. Mirror them here manually:
- Go `ConversationResponse` ‚Üí `web/packages/types/src/api/conversation.ts`
- Go `MessageResponse` ‚Üí `web/packages/types/src/api/message.ts`
- Go `PartnerResponse` ‚Üí `web/packages/types/src/api/partner.ts`

Do this **at the start of each api phase** that adds new endpoints ‚Äî not at the end.
