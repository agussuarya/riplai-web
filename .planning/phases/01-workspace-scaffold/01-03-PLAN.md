Threats considered for this scaffold plan:
- **Client-side secret exposure:** Only `NEXT_PUBLIC_*` variables may be referenced in app code. Do not introduce private environment variable names or secrets.
- **Backend outage fragility:** Health-card fetch failures must render a fallback page, not crash the route.
- **Auth bypass confusion:** Protected layouts are stubs only. Do not implement fake security; label protected layout placeholders as scaffolds for later auth phases.
- **Naming drift:** New app code must use `partner` internally and Indonesian business-friendly public labels. Do not introduce `merchant` files, package names, or type names.
</threat_model>
